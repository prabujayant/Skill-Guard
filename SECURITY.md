# Security Policy

## Responsible Disclosure

If you discover malicious skills using SkillGuard, please follow responsible disclosure practices.

### For Skills Found in Public Repositories

1. **Do NOT** publicly disclose the malicious skill immediately
2. Contact the repository owner/maintainer first
3. Allow 90 days for remediation before public disclosure
4. If no response, report to the hosting platform (GitHub, etc.)

### For Newly Discovered Malware Patterns

1. Report to us at: security@skillguard.example.com
2. Include:
   - Description of the pattern
   - Sample code (anonymized)
   - Detection method
3. We will add detection rules and credit your discovery

## Reporting Vulnerabilities in SkillGuard

If you find a security vulnerability in SkillGuard itself:

1. **Do NOT** open a public issue
2. Email: security@skillguard.example.com
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

We aim to respond within 48 hours.

## Scope

This policy applies to:
- The SkillGuard codebase
- Skills analyzed using SkillGuard
- The SkillGuard dataset

## Out of Scope

- Vulnerabilities in dependencies (report to upstream)
- Social engineering attacks
- Denial of service against our infrastructure

## Safe Harbor

We will not pursue legal action against researchers who:
- Follow this responsible disclosure policy
- Make good faith efforts to avoid privacy violations
- Do not exploit vulnerabilities beyond proof of concept
